## 1. アーキテクチャ

### 前回設計との違い

| 項目 | REST API 版 | Inertia 版 |
|---|---|---|
| フロントエンド | 独立 React SPA | Laravel が配信する React ページ |
| 通信 | JSON REST API | Inertia プロトコル（XHR + JSON） |
| CORS 設定 | 必要 | 不要（同一オリジン） |
| ルーティング | 二重管理 | Laravel routes/web.php のみ |
| 認証 | JWT など別途 | Laravel セッション（そのまま） |
| ファイル URL | signed URL など | 通常の named route |

```
Browser
  └─ Inertia.js (React adapter)
        │  初回: HTML フルレスポンス
        │  以降: XHR + JSON (Inertia プロトコル)
        │  ファイル: multipart POST（Inertia form helper）
        ▼
Laravel 11
  ├─ routes/web.php          ← Inertia ページルート
  ├─ Controllers             ← Inertia::render() を返す
  ├─ Models / Eloquent
  └─ Storage (audio files)
```

## 5. データモデル

### 5-1. テーブル定義

**songs**
```
id              bigint PK
title           string
bpm             unsignedSmallInteger  default: 120
beats_per_measure unsignedTinyInteger default: 4
created_at / updated_at
```

**audio_tracks**
```
id              bigint PK
song_id         FK → songs.id (cascade delete)
name            string              例: "Guitar"
file_path       string              storage/audio/{uuid}.webm
duration_ms     unsignedInteger
offset_ms       unsignedInteger     default: 0
created_at / updated_at
```

**drum_tracks**
```
id              bigint PK
song_id         FK → songs.id (unique, cascade delete)
pattern         json                DrumMeasure[]
created_at / updated_at
```

### 5-2. pattern JSON スキーマ

```json
[
  {
    "measure": 1,
    "notes": [
      {
        "id": "uuid",
        "drumKey": "SN",
        "beat": 1,
        "subdivision": 0,
        "duration": "8"
      }
    ]
  }
]
```

### 5-3. ドラムパート定義

| drumKey | パート | VexFlow pitch | ノートヘッド | Tone.js Synth |
|---|---|---|---|---|
| BD | Bass Drum | a/4 | ● | MembraneSynth (low) |
| SN | Snare | c/5 | ● | NoiseSynth |
| HT | High Tom | e/5 | ● | MembraneSynth (mid-hi) |
| FT | Floor Tom | g/5 | ● | MembraneSynth (mid-low) |
| HH | Hi-Hat | a/5 | × | MetalSynth (short) |
| CY | Crash | b/5 | × | MetalSynth (long) |

## 7. コントローラー設計

### SongController

```php
// index: 曲一覧を Props で渡す
public function index(): Response
{
    return Inertia::render('Songs/Index', [
        'songs' => Song::latest()->get(['id', 'title', 'bpm', 'updated_at']),
    ]);
}

// edit: 全トラックをまとめて Props に乗せる
public function edit(Song $song): Response
{
    return Inertia::render('Songs/Edit', [
        'song'        => $song,
        'audioTracks' => $song->audioTracks,
        'drumTrack'   => $song->drumTrack,
    ]);
}

// store: 曲作成 → エディターへリダイレクト
public function store(StoreSongRequest $request): RedirectResponse
{
    $song = Song::create($request->validated());
    return to_route('songs.edit', $song);
}
```

### AudioTrackController

```php
// store: 録音 Blob を受け取り保存
public function store(StoreAudioTrackRequest $request, Song $song): RedirectResponse
{
    $path = $request->file('audio')->store('audio', 'local');

    $song->audioTracks()->create([
        'name'        => $request->input('name'),
        'file_path'   => $path,
        'duration_ms' => $request->input('duration_ms'),
        'offset_ms'   => $request->input('offset_ms', 0),
    ]);

    return back();
}

// stream: 音声ファイルを返す
public function stream(AudioTrack $audioTrack): StreamedResponse
{
    return Storage::disk('local')->response($audioTrack->file_path);
}
```

### DrumTrackController

```php
// upsert: パターンを JSON で上書き保存
public function upsert(Request $request, Song $song): RedirectResponse
{
    $song->drumTrack()->updateOrCreate(
        ['song_id' => $song->id],
        ['pattern' => $request->input('pattern')]
    );

    return back();
}
```

## 11. 録音フロー詳細

```
1. [● 録音] クリック
   - Tone.start() (AudioContext 開始)
   - navigator.mediaDevices.getUserMedia({ audio: true })
   - MediaRecorder.start()
   - Tone.Transport.start() → ドラムと同期
   - タイマー開始（経過時間表示）

2. [■ 停止] クリック
   - MediaRecorder.stop()
   - Tone.Transport.stop()
   - ondataavailable → Blob 取得

3. アップロード
   const fd = new FormData();
   fd.append('audio', blob, 'recording.webm');
   fd.append('name', trackName);
   fd.append('duration_ms', durationMs);
   fd.append('offset_ms', 0);
   router.post(route('audio-tracks.store', songId), fd, { forceFormData: true });

4. back() でページ再レンダリング → 新トラックが Props に含まれる
5. Waveform.jsx で /audio/{id} から fetch → decodeAudioData → Canvas 描画
```

## 12. 同期再生フロー詳細

```
1. [▶ 再生] クリック
2. useAudioEngine.play({ audioTracks, drumPattern, bpm })

   a. ドラム
      - Tone.getContext().transport.bpm.value = bpm
      - パターンを Tone.Part に変換
      - each note: part.add(time, { drumKey, duration })

   b. 録音音声
      - audioTracks.forEach(track => {
          fetch(route('audio.stream', track.id))
          → arrayBuffer → AudioContext.decodeAudioData → AudioBuffer
          source = AudioContext.createBufferSource()
          source.buffer = audioBuffer
          source.connect(AudioContext.destination)
          Tone.Transport.schedule(t => source.start(t), track.offset_ms / 1000)
        })

   c. Tone.Transport.start()
```

## 13. ドラム打ち込み D&D 詳細

```
[NotePalette]
  draggable div: dragstart → dataTransfer.setData('duration', '4')

[DrumStaff]
  SVG の上に重ねた透明 div (position: absolute, pointer-events: all)
  dragover: preventDefault()
  drop:
    1. duration = dataTransfer.getData('duration')
    2. y座標 → drumKey に変換（drumMap の Y範囲テーブルで逆引き）
    3. x座標 → beat, subdivision に変換
       beat = Math.floor(x / staffWidth * beats_per_measure)
    4. setDrumPattern(prev => [...prev, newNote]) で React state 更新
    5. useAudioEngine.triggerPreview(drumKey) → Tone.js 単発発音
    6. VexFlow 再描画（useEffect の deps に drumPattern）
```

## 15. 制約・注意事項

| 項目 | 内容 |
|---|---|
| AudioContext 制約 | ユーザー操作後に `Tone.start()` を呼ぶ。録音・再生ボタンで初期化 |
| getUserMedia | localhost か HTTPS でのみ動作。Docker 開発環境は http://localhost で可 |
| 録音フォーマット | Chrome: `audio/webm;codecs=opus`、Safari: `audio/mp4`。MIME を Content-Type に記録する |
| Inertia ファイルアップロード | `router.post()` に `{ forceFormData: true }` を付ける。`useForm` は Blob 非対応 |
| VexFlow D&D | SVG に直接 drop イベントを設定せず、上に重ねた div で受け取る |
| ドラムパターン保存タイミング | 自動保存（debounce 2秒）か明示的 [保存] ボタン。プロトタイプは明示ボタン |
| 大容量ファイル | php.ini の `upload_max_filesize` / `post_max_size` を引き上げる（目安: 50M） |
